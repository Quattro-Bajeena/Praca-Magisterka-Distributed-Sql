# ShardingSphere

Apache ShardingSphere  
Apache Software Foundation  
Pochodzenie: Chiny

Middleware / proxy nad SQL  
Nie klasyczna baza danych

OLTP  
Sharding  
Read/write splitting  
Distributed transactions  
Nie OLAP

Kompatybilność: MySql, PostgreSQL

Backendy: MySQL, PostgreSQL, openGauss, Oracle, SQL Server

Architektura: pluggable, micro-kernel, reguły, metadane  
Komponenty: ShardingSphere-JDBC, ShardingSphere-Proxy

Cel: skalowanie i rozszerzanie istniejących baz bez zmiany aplikacji
